### A. 结构化知识框架 (Module 8: Learning by Recording Cases)

**8.0 “通过记录案例学习”简介**

- **定位**:
    
    - 这是课程中第一个关于“学习”的主要主题 。
        
    - 也是“类比推理” (Analogical Reasoning) 的第一个主题，被认为是认知的一个核心过程 。
        
- **核心思想**:
    
    - 智能体在其记忆中记录（存储）过去的具体案例（经验）。
        
    - 当遇到一个新问题时，智能体通过寻找与新问题
        
        **最相似 (most similar)** 或**最接近 (most closely resembling)** 的旧案例，并直接复用该旧案例的解决方案 。
        
- **与推理的关系**: 这种学习方法强调了记忆的重要性，有时记忆可以直接提供答案，从而减少了从头开始进行复杂推理的需求 。
    

**8.1 案例检索方法：最近邻方法 (Nearest Neighbor Method)**

- **目标**: 将“最相似”这个模糊概念操作化、明确化。
    
- **流程**:
    
    - 1.
        
        **表示 (Representation)**: 将所有案例和新问题在一个多维空间中表示出来。每个维度代表一个相关的特征（如积木的宽度、高度） 。
        
    - 2.
        
        **计算距离 (Calculate Distance)**: 计算新问题在空间中的点与所有旧案例的点之间的距离。常用的距离度量是**欧几里得距离 (Euclidean distance)** 。
        
    - 3.
        
        **选择 (Selection)**: 选择距离新问题最近的那个案例，即“最近邻” 。
        
    - 4. **应用 (Application)**: 将最近邻案例的解决方案（如颜色“黑色”）直接应用到新问题上。
            

**8.2 复杂案例与K-最近邻 (KNN)**

- **挑战**: 现实世界的问题通常不能仅用一两个维度来描述。例如，导航问题需要同时考虑起点和终点，每个点又有X和Y两个坐标，这就构成了一个更高维的空间 。
    
- **多维空间中的最近邻**:
    
    - 欧几里得距离公式可以被推广到任意K维空间，这就是
        
        **K-最近邻 (KNN)** 方法的基础 。
        
    - 通过在多维空间（如同时考虑起点X、起点Y、终点X、终点Y的四维空间）中计算距离，可以更准确地找到整体上最相似的案例 。
        
- **局限性**:
    
    - **高维问题**: 当维度非常高时，找到最近邻的计算可能变得不那么简单。
        
    - **直接应用的风险**: 即使找到了最近的案例，也不能保证其解决方案可以**不加修改地**直接应用于新问题 。这个问题引出了下一课的主题：案例推理 (Case-Based Reasoning)。
        

**8.3 认知连接 (The Cognitive Connection)**

- **与人类认知的关系**:
    
    - 人类智能体生活在一个充满
        
        **重复模式**的世界中，我们每天遇到的许多问题（如系鞋带、编程）都与过去经历的问题相同或非常相似 。
        
    - 在这些情况下，我们通常不是从头推理，而是
        
        **让记忆直接提供答案**。这体现了“我们思考得并没有我们想象的那么多” (We don't think as much as we think we do) 的观点 。
        
- **对认知架构的意义**:
    
    - 这种学习方式改变了我们对认知架构三要素（推理、学习、记忆）重要性的看法。
        
    - 它强调了
        
        **学习**（记录案例）和**记忆**（存储和检索案例）的核心作用，它们可以分担甚至替代一部分**推理**的工作，从而实现高效决策 。

### B. 文本思维导图 (Module 8)

- **Module 8: Learning by Recording Cases**
    
    - **核心定义**
        
        - _是什么_: 一种通过存储和复用过去具体经验来进行学习的方法。
            
        - _流程_: 遇到新问题 -> 检索最相似的旧案例 -> 直接应用旧案例的解 。
            
    - **关键机制：案例检索 (Case Retrieval)**
        
        - _方法_: 最近邻方法 (Nearest Neighbor Method)
            
        - _步骤_:
            
            - 1.
                
                **表示**: 将案例和问题映射到多维特征空间 。
                
            - 2.
                
                **计算**: 使用**欧几里得距离**等度量计算点之间的距离 。
                
            - 3. **选择**: 找到距离最近的案例。
                    
    - **处理复杂问题**
        
        - _挑战_: 现实问题维度高（如导航问题需考虑起点和终点）。
            
        - _解决方案_: 将最近邻方法推广到K维空间 (KNN) 。
            
        - _局限性_:
            
            - 高维空间计算复杂。
                
            - 旧的解决方案不一定能直接适用于新问题 。
                
    - **认知连接**
        
        - _与人类认知_:
            
            - 世界充满重复模式，人类大量依赖经验解决日常问题（如系鞋带） 。
                
            - **记忆直接提供答案**，减少了推理的负担 。
                
        - _对认知架构的意义_:
            
            - 突显了
                
                **学习**和**记忆**在智能中的核心作用，而不仅仅是推理 。
