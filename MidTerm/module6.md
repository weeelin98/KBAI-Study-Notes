### A. 结构化知识框架 (Module 6: Production Systems)

**6.0 认知架构 (Cognitive Architectures) 简介**

- **认知智能体的功能**: 可被描述为一个函数，它将一个**感知的历史 (perceptual history)** 映射到一个**行动 (action)** 。核心任务是选择行动。
（简单来说，“映射”就是一种 **“对应关系”**。
你可以把这个函数想象成一个巨大的规则手册或一个两列表格 。
- **第一列 (输入)**: 是智能体到目前为止感知到的一切事物的历史记录 (perceptual history)。例如：“（1秒前）看到球飞过来，（0.5秒前）听到有人喊‘接住’”。
- **第二列 (输出)**: 是针对这一套特定的感知历史，智能体应该采取的确切行动 (action)。例如：“伸出双手去接球”。
“映射”就是指从第一列的任何一种情况，都能准确地对应到第二列的某一个行动的过程。这个“函数”或“映射”本身，就是智能体的“智能”所在——即它内部的决策机制。它的核心任务就是根据它所感知到的一切，来选择当下最应该做的行动 (action selection) 。
所以，"一个认知智能体是将感知的历史映射到一个行动的函数"，这句话的意思是：**一个智能体的本质，就是它内部拥有一套机制，这套机制能根据它过去和现在所感知到的所有信息，最终决定并执行一个具体的动作。**）
    
- **认知架构的层次**:
    
    - **任务/知识层 (Task/Knowledge Level)**: 关注智能体必须做什么任务，以及它拥有什么知识 。
        
    - **算法/符号层 (Algorithm/Symbol Level)**: 关注用于实现任务的算法和知识表示（如语义网络、手段-目的分析） 。
        
    - **硬件/实现层 (Hardware/Implementation Level)**: 关注算法和表示所运行的物理硬件（如大脑、芯片） 。
        
    - **关系**: 上层为下层提供“内容”，下层为上层提供“实现” 。
        
- **认知架构的基本假设**:
    
    - 智能体是目标导向的。
        
    - 生存在丰富、复杂、动态的环境中。
        
    - 使用知识来实现目标。
        
    - 知识以符号形式在特定抽象层级上表示。
        
    - 行为灵活，依赖于环境。
        
    - 能从经验中学习。
        
- **核心公式**: **架构 (Architecture) + 内容 (Content) = 行为 (Behavior)** 。
    
    - 类似于计算机架构，固定架构，通过改变内容（程序/知识）来产生不同行为。
        

**6.1 产生式系统 (Production Systems) 作为认知架构**

- **定义**: 一种认知架构，其知识主要以“如果-那么” (if-then) 规则的形式表示 。这些规则被称为
    
    **产生式规则 (production rules)**。
    
- **SOAR架构示例**:
    
    - **组成**:
        
        - **长期记忆 (Long-Term Memory)**: 存储知识，变化缓慢。包含：
            
            - **程序性知识 (Procedural Knowledge)**: 如何做某事，以产生式规则形式存储 。
                
            - **语义知识 (Semantic Knowledge)**: 关于世界的概念和模型（如概念的框架表示） 。
                
            - **情景知识 (Episodic Knowledge)**: 关于特定事件的记忆（如“我昨天晚餐吃了什么”） 。
                
        - **工作记忆 (Working Memory)**: 存储当前的目标和感知，内容变化迅速 。
            
    - **决策周期 (Decision Cycle)**:
        
        - 1. 当前目标和感知存在于工作记忆中 。
            
        - 2. 工作记忆的内容与长期记忆中程序性知识（产生式规则）的“如果”部分进行匹配 。
            
        - 3. 匹配成功的规则被“触发” (fire) 。
            
        - 4. 规则的“那么”部分（结果）被执行，可能会修改工作记忆（如设定新目标），或提出一个行动算子 。
            
        - 5. 这个过程持续进行，直到一个与运动系统相关的算子被唯一选定，从而产生外部行动 。
            

**6.2 学习机制：组块 (Chunking)**

- **僵局 (Impasse)**: 当决策周期无法做出唯一决策时发生。原因有两种：
    
    - 1. 没有规则可以匹配当前情况（知识不足）。
            
    - 2. 多个规则被触发，提出了多个冲突的行动算子，系统无法选择 。
        
- **组块 (Chunking)**: SOAR中用于解决僵局的学习机制 。
    
- **流程**:
    
    - 1. **触发**: 僵局的发生是组块过程的触发器。
            
    - 2.
        
        **检索**: 系统会查询**情景记忆**，寻找与当前僵局局面（尤其是导致冲突的感知）相似的过去经历 。
        
    - 3.
        
        **学习**: 系统分析过去经历的结果（如，上次对这个击球手投快速球导致了本垒打），并从中**生成一条新的产生式规则** 。
        
    - 4.
        
        **应用**: 这条新规则被添加到程序性知识中，其作用是打破类似的僵局。例如，新规则可能是：“如果快速球和曲线球算子都被建议，且击球手是Parra，则否决快速球算子” 。
        
- **与KBAI三大过程的关联**: 组块过程完美体现了推理、记忆（程序性、情景性）和学习的统一。
    

**6.3 KBAI中的学习基本原理**

- **学习的五个问题**: 如何学 (How)？学什么 (What)？学习来源 (Source)？何时学 (When)？为何学 (Why)？
    
- **KBAI的立场**: 先建立一个**推理理论**，这个理论将决定何时、为何以及学什么，然后才去解决如何学习的问题 。
    
    - **推理优先**: 推理的需求（如解决僵局）驱动了学习的发生。

### B. 文本思维导图 (Module 6)

- **Module 6: Production Systems**
    
    - **认知架构 (Cognitive Architectures)**
        
        - _功能_: 感知历史 -> 行动
            
        - _层次_: 任务/知识层 -> 算法/符号层 -> 硬件层
            
        - _核心公式_: 架构 + 内容 = 行为
            
    - **产生式系统 (Production Systems)**
        
        - _定义_: 以产生式规则 (if-then) 为核心的认知架构
            
        - _SOAR 架构示例_
            
            - _组件_:
                
                - **长期记忆**:
                    
                    - _程序性知识_ (规则)
                        
                    - _语义知识_ (概念)
                        
                    - _情景知识_ (事件)
                        
                - **工作记忆**:
                    
                    - _当前感知_
                        
                    - _当前目标_
                        
            - _决策流程_:
                
                - 工作记忆与规则的 "if" 部分匹配
                    
                - 触发规则
                    
                - 执行 "then" 部分，可能修改工作记忆或选择行动
                    
                - 循环直至选定唯一行动
                    
    - **学习机制：组块 (Chunking)**
        
        - _触发条件_: 僵局 (Impasse)
            
            - _原因_: 无规则匹配 或 多规则冲突
                
        - _过程_:
            
            - 1. 僵局发生
                    
            - 2. 检索情景记忆中的相似经历
                    
            - 3. 分析经历结果，生成一条新的产生式规则来解决冲突
                    
            - 4. 将新规则存入程序性知识中以备后用
                    
        - _体现_: 推理、记忆、学习三位一体
            
    - **KBAI学习观**
        
        - _核心_: 推理优先。由推理的需求（如解决僵局）来决定学习的内容、时机和动机 。
